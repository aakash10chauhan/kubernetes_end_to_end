

# Kubernetes Practical Steps Guide

Using Kubernetes and the kubectl CLI.

---

# 1️⃣ Create Namespace

Apply the namespace configuration:

```bash
kubectl apply -f namespace.yml
```

Check the namespaces:

```bash
kubectl get ns
```

---

# 2️⃣ Create Pod

Create a pod using the YAML file:

```bash
kubectl apply -f pod.yml
```

Check all pods:

```bash
kubectl get pods
```

Check pods in the **nginx namespace**:

```bash
kubectl get pods -n nginx
```

---

# 3️⃣ Access the Pod (Container Shell)

To get inside the pod container:

```bash
kubectl exec -it pod/nginx-pod -n nginx -- bash
```

Now you are **inside the pod**.

Run the following command:

```bash
curl 127.0.0.1
```

This will test the nginx server running inside the container.

---

# 4️⃣ Describe the Pod

To see detailed pod information:

```bash
kubectl describe pod/nginx-pod -n nginx
```

---

# 5️⃣ Deploy Using Deployment

Now create a deployment:

```bash
kubectl apply -f deploy.yml
```

Check deployment:

```bash
kubectl get deployment -n nginx
```

Check pods created by deployment:

```bash
kubectl get pods -n nginx
```

---

# 6️⃣ Scale the Deployment

Increase the number of replicas:

```bash
kubectl scale deployment/nginx-deployment -n nginx --replicas=5
```

Check deployments:

```bash
kubectl get deployments -n nginx
```

Detailed view:

```bash
kubectl get deployments -n nginx -o wide
```

---

# 7️⃣ Perform Rolling Update

Update the nginx image:

```bash
kubectl set image deployment/nginx-deployment -n nginx nginx=nginx:1.27.3
```

Check pods again:

```bash
kubectl get pods -n nginx
```

This process is called **Rolling Update**, because changes happen gradually instead of updating everything at once.
Rolling updates are available only in **Deployments**.

---

# 8️⃣ Check Rollout Status

Verify deployment rollout status:

```bash
kubectl rollout status deployment -n nginx
```

---

# 9️⃣ Delete the Deployment

Delete everything created by the deployment file:

```bash
kubectl delete -f deploy.yml
```

---

# 🔟 Create ReplicaSet

Apply ReplicaSet configuration:

```bash
kubectl apply -f replica.yml
```

Important:

ReplicaSets **do not support rolling updates or rollbacks** like deployments.

Check ReplicaSets:

```bash
kubectl get replicasets -n nginx
```

Check pods:

```bash
kubectl get pods -n nginx -o wide
```

---

# 1️⃣1️⃣ DaemonSet

A **DaemonSet** ensures that **one pod runs on each worker node**.

Example:

If there are **3 worker nodes**, it automatically creates **3 pods**.

You **do not need to define replicas**.

---

# 1️⃣2️⃣ Create Service

Apply the service configuration:

```bash
kubectl apply -f servie.yml
```

Check all resources:

```bash
kubectl get all -n nginx
```

Check services:

```bash
kubectl get service -n nginx
```

---

# 1️⃣3️⃣ Access Application from Browser

Use port forwarding:

```bash
kubectl port-forward service/nginx-service -n nginx 80:80 --address=0.0.0.0
```

If it does not work:

```bash
sudo -E kubectl port-forward service/nginx-service -n nginx 80:80 --address=0.0.0.0
```

Now you can access the application from your **browser using port 80**.

---

# 1️⃣4️⃣ Run Port Forward in Background (Optional)

Run it in the background:

```bash
kubectl port-forward service/nginx-service -n nginx 80:80 --address=0.0.0.0 &
```
